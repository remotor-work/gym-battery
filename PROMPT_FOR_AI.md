# Создай приложение GYM BATTERY

## Кто ты

Ты — разработчик с полной творческой свободой в выборе дизайна деталей и микровзаимодействий. Твоя задача — создать мобильное веб-приложение для тренировок в зале, опираясь на описанную ниже концепцию, бизнес-логику и технические решения. Всё, что не зафиксировано как «обязательное», — на твоё усмотрение.

## 1. ПРОБЛЕМА И ИДЕЯ

### Боль пользователя
В загруженном тренажёрном зале оборудование часто занято. Классические приложения дают жёсткий план: «делай жим штангой на горизонтальной скамье». Скамья занята — стоишь и ждёшь 10 минут, теряешь темп и мотивацию.

### Решающая идея
**У каждого упражнения есть 7 вариантов выполнения** — от штанги до тренажёра и веса тела. Все варианты нагружают одну и ту же мышечную группу, но используют разное оборудование. Если штанга занята — одним тапом переходишь на гантели или тренажёр. Тот же мускул, ноль ожидания.

### Метафора
Тренировка «заряжает батарею» от 0% до 100%. Каждое выполненное упражнение добавляет заряд. Когда все 6 упражнений дня выполнены — батарея полностью заряжена. Это создаёт ощущение прогресса и геймификации.

## 2. ОБЯЗАТЕЛЬНЫЕ ТЕХНИЧЕСКИЕ РЕШЕНИЯ

### Стек — строго фиксирован
- **Единый файл `index.html`** (~4000–5000 строк) — весь HTML, CSS и JS в одном файле. Никаких фреймворков, никакого бандлера.
- **Ванильный JS** (ES6+). Три глобальных синглтона: `App`, `UI`, `Workout`.
- **CSS Custom Properties** для темизации. Все цвета, радиусы — через переменные в `:root`.
- **Делегирование событий** через атрибуты `data-action` на корневом элементе вместо inline-обработчиков.
- **localStorage** для персистентности. Ключ: `'gb_v3'`. Сохранение после каждого действия.
- **Service Worker** (файл `sw.js`) — агрессивное кэширование, Cache First стратегия. При каждом деплое поднимать версию CACHE в sw.js.
- **PWA**: `manifest.json`, иконки, `meta name="apple-mobile-web-app-capable"`.

### Визуальный дизайн — фиксирован
- **Тёмная тема**, mobile-first (max-width: 480px)
- **Шрифты Google Fonts**:
  - `Orbitron` — заголовки, проценты, числа, лейблы (технический «приборный» вид)
  - `Exo 2` — основной текст, описания, кнопки
- **Цветовая палитра** (CSS-переменные в `:root`):
  ```css
  --bg: #161618;      /* основной фон */
  --bg2: #1d1d20;     /* карточки, панели */
  --bg3: #252528;     /* ховеры */
  --bg4: #2e2e33;     /* активные состояния */
  --border: #2c2c32;
  --border2: #3a3a42;
  --text: #a8a8ba;
  --text2: #d0d0e0;
  --text-dim: #7a7a8c;
  --text-off: #5a5a6c;
  --led: #d97320;         /* основной оранжевый акцент */
  --led-dim: #7a3e0e;
  --led-glow: rgba(217,115,32,0.2);
  --led-bg: rgba(217,115,32,0.08);
  --green: #4caf50;
  --red: #f44336;
  ```
- **Язык интерфейса**: русский. Все тексты, кнопки, подсказки — на русском.

### Архитектура экранов (SPA)
Навигация через `UI.showScreen('screen-id')` — один активный `.screen` в любой момент.
```
screen-home → screen-muscles → screen-workout → screen-variation
                                                      ↓
                                           screen-muscles (после завершения)
screen-home → screen-stats (статистика)
screen-home → screen-settings (настройки)
screen-exercise — вспомогательный (обзор вариантов, открывается отдельно)
```

## 3. ОБЯЗАТЕЛЬНЫЕ ТРЕБОВАНИЯ (не менять)

### Язык интерфейса
Русский. Все тексты, кнопки, подсказки — на русском.

### Тренировочная программа
Сплит **Upper/Lower** — 4 тренировочных дня:

| День | Фокус | Описание |
|------|-------|----------|
| Д1 — ВЕРХ | Жимы | Грудь горизонтально + под углом, разводка, плечи (средние дельты), трицепс. Тяга вертикальная для баланса. |
| Д2 — НИЗ | Приседания | Присед со штангой (основной компаунд), румынская тяга, жим ногами, сгибание ног, икры, выпады. |
| Д3 — ВЕРХ | Тяги | Тяга горизонтальная + в наклоне, жим плеч, жим груди decline, бицепс, face pull (задние дельты). |
| Д4 — НИЗ | Жимы ног | Жим ногами (основной), болгарские выпады, вариация приседа, разгибание, икры, сгибание ног. |

- **4 дня, 6 упражнений в каждом** — фиксировано.
- Дни **не привязаны к дням недели**. Пользователь тренируется в своём ритме и сам выбирает какой день делать.
- Прогресс дней **сбрасывается в начале новой недели** (при первом открытии в новую неделю, определяется по ISO-номеру недели).

### Структура упражнений

**ДЕНЬ 1 — ВЕРХ (Жимы: грудь, плечи, трицепс)**
1. Жим — Грудь (горизонтально): Основной компаунд с утяжелением
2. Тяга — Спина (вертикально): Компенсирует жим, широчайшие
3. Жим — Грудь (под углом): Верхние пучки, передние дельты
4. Разводка — Грудь: Изоляция, пиковое сокращение
5. Подъём — Плечи (средние дельты): В стороны, боковой подъём
6. Разгибание — Трицепс: Разгибание с различным хватом

**ДЕНЬ 2 — НИЗ (Приседания: квадрицепсы основной)**
1. Присед — Квадрицепс (основной): Присед со штангой, основной компаунд
2. Тяга — Задняя цепь (РДЛ): Румынская, противовес приседу
3. Жим — Квадрицепс (жим): Жим ногами, вариация компаунда
4. Сгибание — Бицепс бедра: Изоляция задней цепи
5. Подъём — Икры: Подъём на носки
6. Выпад — Квадрицепс (вариация): Выпады, дополнительный объём

**ДЕНЬ 3 — ВЕРХ (Тяги: спина, бицепсы, дельты)**
1. Тяга — Спина (горизонтально): Основной компаунд спины
2. Жим — Плечи (над головой): Жим плеч, баланс с тягами
3. Тяга — Спина (в наклоне): Верхняя часть, трапеция
4. Жим — Грудь (decline): Нижние пучки груди, вариация
5. Сгибание — Бицепс: Сгибание, изоляция бицепса
6. Тяга — Задние дельты: Face Pull, тяга к лицу

**ДЕНЬ 4 — НИЗ (Жимы ног: задняя цепь ориентир)**
1. Жим — Квадрицепс (жим): Жим ногами, основной компаунд дня
2. Выпад — Задняя цепь + квад: Болгарский или выпад тяжёлый
3. Присед — Квадрицепс (вариация): Вариант приседа (фронт, гоблет)
4. Разгибание — Квадрицепс (изоляция): Разгибание ног, финишер
5. Подъём — Икры: Подъём на носки, вариация
6. Сгибание — Бицепс бедра: Сгибание ног, дополнительное

### 7 вариантов на каждое упражнение (168 итого)

Для каждого из 24 слотов — **ровно 7 вариантов**. Принцип распределения:

| Позиция | Оборудование | Приоритет |
|---------|-------------|-----------|
| 1–2 | Свободные веса (штанга, гантели) | Высший (★★★★★) |
| 3–4 | Блоки, тросы, тренажёр Смита | Высокий (★★★★☆) |
| 5–6 | Специализированные тренажёры (включая Hammer Strength) | Средний (★★★☆☆) |
| 7 | Минимальное оборудование / вес тела | Базовый (★★☆☆☆) |

Приоритет вычисляется из позиции — не хранится отдельно.

Для каждого варианта нужно:
- **`img`-slug** — kebab-case строка для имени файла изображения (пример: `chest-flat-barbell`, `back-pullup-bodyweight`). Формат: `{мышца}-{движение}-{оборудование}`.
- **Название** на русском (+ английское в скобках)
- **Теги оборудования** — массив строк: что нужно для выполнения
- **Целевые мышцы** — строка
- **Описание техники** — 2–4 предложения, русский

**Обязательные варианты для Д1, упражнение 1 (жим грудь горизонт):**
Включи Hammer Strength тренажёр (ISO-Lateral Chest Press) как один из вариантов 5–6 с тегами `["Hammer Strength", "Блины", "Независимые рычаги"]` и img-slug `chest-flat-hammer`.

## 4. ОСНОВНОЙ FLOW ТРЕНИРОВКИ

### screen-home — Главная
- 4 батарейки-карточки дня в **одну горизонтальную линию** (flexbox, равные колонки).
- Каждая батарейка показывает: метку дня (Д1/Д2/Д3/Д4), мини-прогресс-бар (0–6 выполненных), процент заряда.
- Завершённые дни визуально отмечены (полная батарея, зелёный акцент).
- Навигация: статистика и настройки — из этого экрана.

### screen-muscles — Мышечные группы
- 6 карточек слотов текущего дня.
- Каждая показывает: тип движения, название мышцы, статус (→ выбрать / ✓ выполнено / ⊘ занято / ↺ ожидает).
- Механика «Занято»: текущий вариант откладывается в конец очереди → сразу предлагается следующий по приоритету. Пользователь всегда движется вперёд.

### screen-workout — Тренировка (ДЕЛАЙ СЕЙЧАС)
- Показывает **текущий лучший вариант** для выбранного слота (зона «ДЕЛАЙ СЕЙЧАС»).
- Под основной карточкой — **список альтернатив** (все остальные варианты из очереди, приглушённые компактные карточки). Отдельного экрана «Варианты» нет — всё видно сразу.
- Клик по альтернативе → немедленно переходит на подходы этого варианта.
- На каждой карточке (основной и альтернативах) — **полоска активности (heat bar)**: горизонтальная полоска с CSS-градиентом от красного (никогда не делал) до зелёного (частое использование), ширина = heat × 10%.

### screen-variation — Подходы (ПОДХОДЫ)
- Название, мышца, приоритет (звёзды), кнопки «Лайк» / «Забанить».
- Раздел оборудования (теги).
- **Полоска активности** (heat bar, с лейблом «АКТИВНОСТЬ»).
- Иллюстрация техники (GIF или PNG).
- Кнопки подходов (Подход 1 / 2 / 3). Нажимаются после каждого подхода.
- После каждого подхода — **компактный таймер отдыха в углу экрана** (правый нижний, `position: fixed`), не блокирует основной экран. Размер ~50×50px SVG-кольцо + цифры + кнопка «×» (пропустить).
- **Кнопка «✓ Выполнено»** — для случаев когда забыл нажимать каждый подход. Автоматически засчитывает все невыполненные подходы с текущими weight/reps и завершает упражнение.
- Поля ввода веса (кг) и повторений.
- Авто-заполнение из последней тренировки того же дня.
- Описание техники.

## 5. СИСТЕМА АКТИВНОСТИ (HEAT BAR)

Каждый вариант упражнения имеет уровень активности (`heat`), от 0 до 10.

**Правила:**
- При завершении упражнения — `heat` варианта увеличивается на +1 (макс 10).
- **Ежемесячный decay**: при первом открытии в новый месяц — все ненулевые значения уменьшаются на 1. Это позволяет полоске «остывать» если упражнение давно не выполнялось.
- Хранится в `App.persist.exerciseHeat` как объект `{ "img-slug": число }`.
- Последний месяц применения decay хранится в `App.persist.lastDecayMonth` как строка `"YYYY-MM"`.

**Отображение:**
```
АКТИВНОСТЬ  ████████░░░░  (heat=8, ширина 80%)
```
- Полоска всегда рендерит полный CSS-градиент: `linear-gradient(to right, #e53935, #ff9800 45%, #8bc34a 70%, #00e676)`
- Ширина контейнера (`width`) = `heat * 10%`.
- Показывается на: карточке «ДЕЛАЙ СЕЙЧАС», каждой альтернативной карточке, экране подходов (screen-variation).

## 7. БАН И ЛАЙК

- **Бан (⊘):** исключает вариант из очереди. Доступен на screen-variation и из screen-exercise.
- **Лайк (♥):** при равном приоритете лайкнутые идут первыми.
- Список банов и лайков хранится в `App.persist` (объект `likedVariants`, `bannedVariants` с ключами `"varKey-varIdx"`).
- Разбан доступен в настройках.

App.persist = {

## 12. КРИТЕРИИ ПРИЁМКИ

Готовое приложение должно:

1. Открываться по URL на GitHub Pages.
2. Устанавливаться на телефон как PWA.
3. Работать офлайн (Service Worker).
4. Содержать **168 вариантов упражнений** (24 × 7) с img-slug, тегами оборудования и описанием техники.

7. Показывать альтернативные варианты прямо на экране тренировки (не отдельным экраном).

9. Таймер отдыха — компактный в углу, не перекрывает экран.
10. Сохранять и восстанавливать прогресс при перезагрузке.
11. Показывать историю и статистику.
12. Бан/лайк вариантов работает.
13. Интерфейс полностью на русском языке.
